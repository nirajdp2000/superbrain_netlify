======================================================================
1. WHAT THIS APP REALLY IS
======================================================================

Superbrain India is a Node.js HTTP server plus a React frontend.

The server entry point is:
- src/server.mjs

The frontend is built by Vite from:
- frontend/

The built frontend is served from:
- public/

There are 2 different analysis stacks in this codebase:

1. Primary live analysis stack
- Used by /api/analyze, /api/ask, /api/intelligence, /api/dashboard, /api/news, /api/macro
- Main modules:
  - src/services/analysis-service.mjs
  - src/services/market-service.mjs
  - src/services/news-service.mjs
  - src/services/universe-service.mjs
  - src/services/top-signals-service.mjs
- This is the main engine the current UI relies on.

2. Secondary /api/v2 stack
- Used by /api/v2/* routes
- Main modules:
  - src/services/trading-analysis-engine.mjs
  - src/services/flexible-search-engine.mjs
  - src/services/advanced-universe-service.mjs
- Important: parts of this stack still use heuristic or mock-style data/scoring. Do not describe it as equal in fidelity to the primary live analysis engine.

======================================================================
2. PRIMARY ENDPOINTS AND WHAT THEY DO
======================================================================

/api/health
- Returns service health, default watchlist, and Upstox connection status.

/api/universe
- Searches the stock universe.

/api/analyze
- Runs the main multi-stock analysis engine for one strategy and returns ranked rows.

/api/intelligence
- Runs the main engine for one stock and one strategy, then enriches it with deep reasoning and execution gating.

/api/dashboard
- Builds the homepage/dashboard response using the main engine plus market-wide opportunity scan.

/api/ask
- Maps a natural-language query to a stock and strategy.
- By default it returns all strategy views for that stock, not just one horizon.

/api/news
- Returns either symbol-specific news or the full macro/news bundle.

/api/macro
- Returns market context plus macro and geopolitical headlines.

/api/v2/*
- Separate exploratory engine for flexible search, stock analysis, prediction-style output, and market-signal scans.
- Use with caution because some of these modules are still partially mock-based.

----------------------------------------------------------------------
5A. Quote fetching
----------------------------------------------------------------------

Main function:
- getQuotes(stocks)

Quote source order:
1. Check Upstox status via getUpstoxStatus()
2. If connected, fetch live quotes from Upstox first
3. For any stock not covered by the live map, fall back to Yahoo quote fetch
4. Exception: if the quote request is a large sweep (> 250 stocks), Yahoo fallback is skipped

Upstox quote details:
- Source module: src/services/upstox-service.mjs
- fetchUpstoxQuotes(stocks)
- Batch size: 120 instrument keys
- Batch concurrency: 6
- Quote source labels:
  - UPSTOX_LIVE
  - UPSTOX_LIVE_SHARED

Yahoo quote fallback:
- Based on Yahoo chart metadata, not Level 2/live exchange feed
- Uses fetchYahooChart(symbol, "5d", "1d")
- Quote source label:
  - YAHOO_DELAYED

Important quote caveat:
- Intraday decisions are explicitly penalized or blocked when the quote is not live Upstox data.

NSE logic:
- Calls https://www.nseindia.com/api/quote-equity
- Primarily used for P/E fallback
- Source label:
  - NSE_PUBLIC

Unavailable fundamentals:
- If all public sources fail, the engine still returns a shaped payload with source = UNAVAILABLE and a reason string.

6. TECHNICAL SNAPSHOT ALGORITHM
======================================================================

Main function:
- computeTechnicalSnapshot(candles, quote)

Required minimum history:
- If latest price is missing or daily candles < 30, the technical snapshot returns neutral/fallback values.

Indicators calculated:
- sma20
- sma50
- return20d
- return60d
- nearHighRatio (vs 120-day high)
- drawdown (vs rolling peak)
- volumeSurge (latest volume vs 20-day average)
- rsi14
- macd
- vwap20
- support20
- resistance20
- support60
- resistance60
- higherTimeframe
- regime
- candlestick

Technical score starting point:
- Base score = 50

Technical score adjustments:
- +10 if price > sma20
- +10 if sma20 > sma50
- +10 if return20d > 4
- +8  if return60d > 10
- +8  if nearHighRatio > 0.92
- +6  if volumeSurge > 1.25
- +6  if RSI is between 54 and 72
- -6  if RSI < 40
- +5  if MACD histogram > 0
- -5  if MACD histogram < 0
- +4  if price > vwap20
- -4  if price < vwap20
- -10 if drawdown > 18
- -6  if volatility > 3.2
- -10 if return20d < -5

Technical trendBias from final technical score:
- >= 62 -> BULLISH
- <= 42 -> BEARISH
- else  -> NEUTRAL

Higher timeframe logic:
- Daily candles are aggregated into 5-day buckets to approximate weekly candles
- Last 26 weekly buckets are used
- Higher timeframe trend is bullish when:
  - price > weekly sma4
  - weekly sma4 > weekly sma8
  - 4-week return > 0
- Higher timeframe trend is bearish when the inverse is true

Regime classification:
- HIGH_VOLATILITY if volatility >= 3.8
- TRENDING if:
  - directional strength >= 6
  - trend spread pct >= 2.2
  - 20-day range width pct >= 6
- SIDEWAYS if:
  - 20-day range width pct <= 8
  - abs(return20d) <= 4.5
  - abs(return60d) <= 10
- Else TRANSITIONAL

Trend state inside regime:
- UPTREND
- DOWNTREND
- RANGE

  - Fake breakout
  - Fake breakdown
- Weekly trend bias
- MACD posture
- RSI context

Candlestick scoring:
- Each candidate starts with its own reliability score.
- The score is then adjusted by confluence:
  - alignment with prevailing trend
  - reversal location or continuation location
  - market structure
  - trap confirmation
  - weekly trend agreement/conflict
  - MACD posture
  - RSI position
  - regime suitability
  - volume surge
  - penalty for noise-zone patterns

Candlestick quality tiers:
- score >= 7.6 -> Ultra
- score >= 5.8 -> High
- score >= 3.8 -> Moderate
- else         -> Weak

Candlestick validity:
- Weak patterns are marked Ignore
- Stronger patterns can still be downgraded later if they do not align with the system verdict/context

Trade trigger output:
- The engine also generates a trigger sentence, not just the pattern name.
- Example logic:
  - bullish reversal near support -> wait for follow-through above signal high with risk below signal low
  - bearish reversal near resistance -> inverse logic

Current candlestick limitation:
- The engine is good for daily pattern context, but it is not an intraday candle engine.

----------------------------------------------------------------------
8A. Company-news scoring
----------------------------------------------------------------------

Main summarizer:
- summarizeSymbolNews(symbol, items)

Base score:
- Starts at 50

For up to the top 8 relevant symbol headlines:
- weight = freshnessScore * credibility * verified multiplier * official multiplier
- score increment = (sentiment.score * 18 + tagBias) * weight

Additional outputs:
- verifiedCount
- officialCount
- newsCount
- companySpecificCount
- highCredibilityCount
- realTimeCount
- avgCredibility
- latestPublishedAt
- signalBalance
- freshnessScore
- dominantTags
- sourceCoverage
- bullishHeadlines
- bearishHeadlines

Evidence grade rules:
- A if:
  - official >= 1 and realTime >= 1
  - or verified >= 2 and avgCredibility >= 0.9
- B if:
  - verified >= 1 and highCredibility >= 2
  - or realTime >= 2 and avgCredibility >= 0.88
- C if:
  - newsCount >= 1 and avgCredibility >= 0.82
- D otherwise

----------------------------------------------------------------------

----------------------------------------------------------------------

Main function:
- buildEventExposure(stock, globalNews)

Base score:
- Starts at 50

Inputs:

Event-exposure output:

- Official headline count
- High-credibility headline count

- 18 to 95 after coverage penalty

12. TRADE VALIDITY GATE AND EXECUTION LOGIC
======================================================================

Important distinction:
- A bullish or bearish verdict does not automatically mean "trade now".
- The app has a stricter institutional-style gate in buildTradeDecision().

The engine can still return:
- action = NO_TRADE
- status = WAIT - CONDITIONS NOT MET

Trade gate requires:
- Directional verdict is BUY/SELL class
- Strong multi-factor confluence
- Clear directional bias
- No high-severity conflicts
- Higher timeframe not in conflict
- Market regime is not unfavorable
- Current location versus key levels is acceptable
- Entry timing is optimal or acceptable early
- Signal clarity is not mixed
- Enhancement-layer confluence passedChecks >= 4
- Reward-to-risk >= 2
- No blocking issues

Blocking issues include:
- Missing current price
- Incomplete technical snapshot
- Missing target/stop structure
- Intraday without live Upstox quote

Execution plan output includes:
- entry type (market or limit)
- entry price
- stop loss
- hard stop loss
- invalidation level
- target 1
- target 2
- position-sizing guidance
- trailing stop guidance
- exit plan

14. MARKET-WIDE TOP SIGNALS ENGINE
======================================================================

Main module:
- src/services/top-signals-service.mjs

Purpose:
- Scan a broad universe and surface top bullish and bearish names

Universe source:
- src/services/broad-universe-service.mjs

Broad-universe source order:
1. Upstox BOD instrument dumps
   - NSE.json.gz
   - BSE.json.gz
2. Disk cache:
   - data/broad-equity-universe.json
3. Local universe fallback if remote source fails

Broad-universe filters:
- Keeps only equity instruments
- Excludes ETF, ETN, MF, bond-like instruments, rights-style records, and other non-standard listings

Scan flow:
1. Fetch broad universe
2. Fetch quotes for whole universe
3. Require minimum quote coverage:
   - min(250, max(25, round(universe_size * 0.02)))
4. Build quick ranks
5. Build bullish and bearish shortlists
6. Run deep analyzeMarket() only on shortlisted stocks
7. Attach pre-scan metadata to deep-analysis results

Pre-scan quickScore:
- Uses changePct
- open-to-current move
- current position within day range
- weights vary by strategy

Additional shortlist filters:
- tradeability score
- turnover
- liquidity floor
- anti-exhaustion checks

Important limitation:
- If broad quote coverage is too low, the service throws:
  "Broad market quote sweep is unavailable. Reconnect Upstox to restore 5000+ stock scanning."

18. SHORTEST ACCURATE SUMMARY FOR ANOTHER AI
======================================================================

If you need a compressed description, use this:

Superbrain India is primarily a multi-factor Indian equities analysis engine built around analysis-service.mjs. For each stock it resolves a symbol, fetches quote + daily candles + fundamentals, builds a technical snapshot, summarizes company news, scores macro/event exposure, then combines everything with strategy-specific weights for intraday, swing, position, and longterm horizons. The main live quote source is Upstox, with Yahoo delayed fallback for smaller requests; daily candles come from Yahoo; fundamentals come from Screener first and then Moneycontrol/NSE fallback; macro context comes from Yahoo benchmark instruments plus NSE FII/DII data; company and macro news come from RSS feeds plus Google News RSS, with cross-source verification and evidence grading. The main engine also adds candlestick context on the daily chart with weekly trend confirmation, conflict detection, execution gating, and a no-trade state when confluence is not strong enough. Separate /api/v2 modules exist, but some of them still rely on mock or placeholder scoring/data and should not be described as the same fidelity as the primary analysis stack.
